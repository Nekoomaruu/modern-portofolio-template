# Contributing

Thanks for your interest in improving this template! Contributions of all kinds are welcome — bug reports, new features, docs improvements, or design polish.

## Quick start

1. Fork the repo and clone your fork.
2. Create a branch: `git checkout -b feat/your-feature`.
3. Install deps: `npm install`.
4. Run dev server: `npm run dev`.
5. Make your changes.
6. Commit with a clear message: `git commit -m "feat: add X"`.
7. Push and open a Pull Request.

## Guidelines

- **Keep it minimal.** This template's value is restraint — please don't add heavy dependencies without a strong reason.
- **Use design tokens.** No hardcoded colors (`text-white`, `bg-[#000]`). Edit `src/index.css` instead.
- **Respect both themes.** Test changes in dark *and* light mode.
- **Stay accessible.** Use semantic HTML, alt text on images, keyboard-reachable interactive elements.
- **Type things.** This is TypeScript — please don't `any` your way out.

## Reporting bugs

Open an issue with:

- What you expected to happen
- What actually happened
- Steps to reproduce
- Screenshots / browser / OS if relevant

## Feature requests

Open an issue first to discuss the idea before opening a PR for any non-trivial feature. Helps avoid wasted work 💚

## Code of Conduct

Be kind. Be respectful. Assume good intent. Harassment of any kind is not tolerated.
